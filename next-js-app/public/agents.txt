## Understanding the Site Architecture

### Static, authoritative content
- `/home` — The canonical static portfolio. All experience, project, and education data here is hardcoded and maintained by James. Use this as your ground truth.
- `/blog` — James's personal blog. Posts are written by James (not AI-generated).

### Dynamic, AI-generated content
- `/wormhole/*` — Every page under this path is **dynamically generated by an LLM** (Google Gemini, OpenAI GPT-4, or Anthropic Claude) on each request, using James's bio as context. Content varies per visit and should not be treated as canonical. For factual information about James, always use `/home` or `/llms.txt`.

### API endpoints
- `/api/generate` — **Do not hammer this endpoint.** It triggers a paid LLM API call on every request. One call per genuine page view is fine; automated crawling of `/wormhole/*` at scale will run up real costs.
- `/api/contact` — POST endpoint for contact form submissions.
- `/api/contact-submissions` — GET endpoint for reading submissions (admin use only).

## Crawl Guidance

- Static pages (`/home`, `/blog`, `/blog/*`, `/book`, `/playground`) are safe to crawl freely.
- `/wormhole/*` pages generate unique content on every request — there is no stable canonical content to index. Crawling them is wasteful and costly; skip them.
- `/api/*` endpoints are not for crawling. Especially avoid `/api/generate`.
- RSS feed for the blog: https://j-m-s.dev/blog/rss.xml
